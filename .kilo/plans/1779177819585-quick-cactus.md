# Switch PowerSync from react-native-quick-sqlite to OP-SQLite

## Current State

- `package.json` already has `@powersync/op-sqlite: ^0.9.8` installed
- `package-lock.json` shows `@journeyapps/react-native-quick-sqlite` as an **optional** peer dependency of `@powersync/react-native` (line 3433)
- `src/lib/powersync-db.ts` uses `new PowerSyncDatabase({ schema, database: { dbFilename: 'ord.db' } })` — this uses the **default** database adapter, which falls back to `react-native-quick-sqlite` when no explicit `database` factory is provided
- `app.json` has no OP-SQLite plugin configured
- The `@op-engineering/op-sqlite` peer dependency is present in `package-lock.json` (line 3367)

## Problem

The `PowerSyncDatabase` instantiation in `src/lib/powersync-db.ts` does not explicitly use the OP-SQLite open factory. When no `database` factory is provided, PowerSync defaults to `react-native-quick-sqlite`.

## Changes Required

### 1. Update `src/lib/powersync-db.ts`

Replace the default database configuration with explicit OP-SQLite factory:

```ts
// Before
import { PowerSyncDatabase, Schema, Table, column } from '@powersync/react-native'

const schema = new Schema({ ... })

export const powerSyncDb = new PowerSyncDatabase({
  schema,
  database: { dbFilename: 'ord.db' },
})

// After
import { PowerSyncDatabase, Schema, Table, column } from '@powersync/react-native'
import { OPSqliteOpenFactory } from '@powersync/op-sqlite'

const schema = new Schema({ ... })

const opSqlite = new OPSqliteOpenFactory({
  dbFilename: 'ord.db',
})

export const powerSyncDb = new PowerSyncDatabase({
  schema,
  database: opSqlite,
})
```

### 2. No package.json changes needed

- `@powersync/op-sqlite` is already in `package.json`
- `@op-engineering/op-sqlite` is already in `package-lock.json` as a transitive/peer dependency
- `@journeyapps/react-native-quick-sqlite` is an **optional** peer dependency — no need to remove it from lock file; it won't be used

### 3. No app.json changes needed

OP-SQLite does not require an Expo plugin entry (unlike quick-sqlite with `use_frameworks!`).

## Verification Steps

1. Run `npx expo install` to ensure peer dependencies are satisfied
2. Run `npx tsc --noEmit` to check for type errors
3. Run `npx expo lint` to check for lint issues
4. Build and test on device/simulator to confirm database works
