# Plan: Add `ready` and `error` to `useDbTest` hook

## Problem
The `useDbTest` hook only returns `items` and `loading`, so callers cannot distinguish:
- An empty table (valid state with 0 rows)
- "No results yet" (watch hasn't fired)
- A watch failure (error state)

## Changes

### 1. `src/hooks/use-db-test.ts` — Add `ready` and `error` state

**Add state variables:**
```ts
const [ready, setReady] = useState(false);
const [watchError, setWatchError] = useState<Error | string | null>(null);
```

**Update the `useEffect` watch (lines 27-43):**
- In `onResult`: call `setReady(true)` alongside `setItems`, and clear any previous error via `setWatchError(null)`
- In `onError`: call `setWatchError(error)` and `setReady(true)` so the hook is no longer in "not-yet-loaded" state
- In cleanup: abort is fine as-is; `ready` won't get stuck because a new mount resets `ready=false` via `useState` initializer

**Update return signature (line 77):**
```ts
return { items, loading, ready, error: watchError, loadItems, insertItem, deleteAll };
```

### 2. `src/app/(tabs)/db-test.tsx` — Consume new return values

**Destructure additional values:**
```ts
const { items, loading, ready, error, loadItems, insertItem, deleteAll } = useDbTest();
```

**Update the empty-state rendering (lines 136-137):**
- If `!ready`: show "Loading..." status text
- If `error`: show error status text
- If `ready && items.length === 0`: show "No items yet" (existing behavior)

## Verification
- TypeScript strict mode passes (no `any`)
- Hook correctly transitions: `ready=false` → `ready=true` on first `onResult` or `onError`
- Cleanup via `abortController.abort()` does not leave `ready` stuck (new mount resets it)
- Screen displays appropriate state for each case
